## 合并文件图例修复

### 问题
合并结果中的图例（HATCH填充图案和INSERT块引用）没有正确复制，导致图例显示不完整。

### 原因
1. **HATCH实体**：复制逻辑不完整，边界路径处理有误
2. **INSERT实体**：块引用复制了，但对应的块定义没有复制到目标文档
3. **LWPOLYLINE等实体**：手动复制逻辑不完善，某些实体类型丢失

### 修复内容
1. **添加`copy_blocks_to_doc()`函数**：在复制实体前，先将源文档中的所有块定义复制到目标文档
2. **重写`copy_entity_with_offset()`函数**：
   - 使用`entity.copy()`和`translate()`原生方法复制大多数实体类型
   - 添加`target_doc`参数用于检查块定义是否存在
   - INSERT块引用特殊处理
3. **重写`copy_entity_to_block()`函数**：
   - 使用`entity.copy()`原生方法复制块定义内部的实体
   - 保留手动复制作为备用方案
4. **添加`copy_entity_manual()`函数**：备用手动复制方法
