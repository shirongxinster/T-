# 2026-03-25 桥台构件支持 + 单柱墩支持 + 批量处理

在 bridge_disease_main_lower.py 中添加了对桥台和单柱墩构件的支持，并实现了批量处理功能：

## 新增功能

### 1. 桥台支持
- **模板配置**：
  - 带台身桥台：模板文件 `带台身桥台.dxf`
  - 不带台身桥台：模板文件 `不带台身桥台.dxf`
- **坐标系统**：
  - 带台身桥台：台帽原点 (66, 245)，台身原点 (66, 219)
  - 不带台身桥台：台身原点 (57, 219)
  - 右下倾斜1度
- **病害处理**：
  - 新增 `process_abutment_disease()` 函数
  - 支持所有病害类型
  - **只处理台身和台帽，跳过前墙、护栏**

### 2. 单柱墩支持
- **模板文件**：`单柱墩.dxf`
- **坐标系统**：
  - 盖梁小桩号面原点：(52, 264)

  - 大桩号面柱子矩形：(385, 233) 到 (409, 194)
  - 右侧面矩形：(178, 130) 到 (202, 70)
  - 内侧面矩形：(76, 130) 到 (100, 70)
  - 右下倾斜1度
- **病害处理**：
  - 新增 `process_single_pier_disease()` 函数
  - 处理盖梁和柱子病害

### 3. 批量处理功能
- **上部程序** (bridge_disease_main_upper.py)：
  - 从 `input` 目录读取所有Excel文件
  - 自动处理每个文件的上部病害
  - 生成：`{Excel文件名}-上部病害.dxf`

- **下部程序** (bridge_disease_main_lower.py)：
  - 从 `input` 目录读取所有Excel文件
  - 自动处理每个文件的下部病害（双柱墩、单柱墩、桥台）
  - 生成：`{Excel文件名}-下部病害.dxf`

### 4. 文本替换
- 路线名称 → 替换模板里的 LLL
- 桥梁名称 → 替换模板里的 QQQ
- 构件编号 → 替换模板里的 GGG（盖梁）/ TTT（桥台）

### 5. 主程序逻辑
- 分别处理三种构件类型：双柱墩、单柱墩、桥台
- 按构件类型和编号生成单独的文件
- 所有文件合并到最终输出文件

   - 从顶部向下绘制

   - 步长改为0.2更精细
3. **添加缺失变量**：在`process_cap_beam_disease`函数中添加scale_x和scale_y定义

**修改文件**：

### 2026-03-25 19:10 修复：蜂窝麻面标注引线起点距离图形太远

**问题**：双柱墩盖梁的蜂窝麻面标注，引线起点距离实际绘制的图形太远。

**示例病害**：
- 盖梁2 大桩号面，x=11～12m，y=0～1m，蜂窝麻面 S=1.00m2
- 盖梁2 小桩号面，x=6～7m，y=0～1m，麻面 S=1.00m2

**原因**：`calculate_start_point()` 函数使用**原始病害坐标**计算引线起点，而蜂窝麻面的 `draw_honeycomb()` 函数返回的实际 bbox 可能因为平铺计算（只画完整格子）而与原始区域不同，导致起点距离实际图形太远。

**修复**：修改 `process_cap_beam_disease()` 函数中蜂窝麻面的处理逻辑：
1. 使用 `draw_honeycomb()` 返回的实际 bbox 计算引线起点
2. 从 bbox 的角点引出（右下角或左上角）
3. 根据 `go_left` 动态选择引出角点

**修改前**：
```python
start_x, start_y = calculate_start_point(disease, origin, other_diseases, go_left=False)
```

**修改后**：
```python
bbox_min_x, bbox_min_y, bbox_max_x, bbox_max_y = bbox
start_x = bbox_max_x  # 默认从右下角引出
start_y = bbox_max_y
# ... 根据 go_left 重新计算 ...
if go_left:
    start_x = bbox_min_x  # 从左上角引出
    start_y = bbox_min_y
else:
    start_x = bbox_max_x  # 从右下角引出
    start_y = bbox_max_y
```

**修改文件**：bridge_disease_main_lower.py 第1498-1534行

### 2026-03-25 19:19 优化：下部病害标注方向智能选择

**问题**：下部病害的所有标注都向右下偏移（315°），没有根据病害位置智能选择方向。

**原因**：`get_safe_angle()` 函数按固定优先级选择角度（315°优先），没有考虑病害的实际位置。

**优化方案**：重写 `get_safe_angle()` 函数，参考上部程序逻辑：
1. **水平方向**：根据病害X中点与构件中心线的关系
   - 病害在中心线左侧 → 引线向右（45°或315°）
   - 病害在中心线右侧 → 引线向左（135°或225°）
2. **垂直方向**：根据病害Y中点与构件垂直中心的关系
   - 病害在上方 → 引线向下（315°或225°）
   - 病害在下方 → 引线向上（45°或135°）

**构件中心线定义**：
- 盖梁：X中心=6.0m，Y中心=0.5m
- 墩柱：X中心=0.5m，Y中心=3.5m
- 台身：X中心=5.0m，Y中心=2.0m
- 台帽：X中心=5.0m，Y中心=0.5m

**角度组合**：
- 右上：45°（病害在左下方）
- 右下：315°（病害在左上方）
- 左上：135°（病害在右下方）
- 左下：225°（病害在右上方）

**修改文件**：bridge_disease_main_lower.py 第742-830行（get_safe_angle函数）
