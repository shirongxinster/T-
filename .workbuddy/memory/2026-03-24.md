## 引线角度逻辑重构（get_label_angle 完全重写）

### 新逻辑（更简洁）
原来的逻辑复杂、有随机性、容易出错。重写为统一的两步规则：

**第一步：水平方向（左/右）**
- T梁水平中心线 X_CENTER = 284（x=84~483，共399单位）
- 病害x中点 < 284 → 引线向右，连接病害图右侧
- 病害x中点 >= 284 → 引线向左，连接病害图左侧

**第二步：垂直方向（上/下，仅腹板）**
- 各腹板有Y轴中心线（基于实际Y坐标范围中点）：
  - 上部左腹板 Y中心 = 273.5
  - 上部右腹板 Y中心 = 245.3
  - 下部左腹板 Y中心 = 161.1
  - 下部右腹板 Y中心 = 132.9
- 病害y中点 > Y中心 → 在上方 → 引线向下
- 病害y中点 <= Y中心 → 在下方 → 引线向上
- 翼缘板/梁底/马蹄：固定方向（左翼缘板向下，右翼缘板向上，马蹄左向上，马蹄右向下）

**组合角度：右上=45°，左上=135°，左下=225°，右下=315°**

### 同期修复
- `get_label_bbox` 不再把引线起点纳入bbox（引线起点在病害图上，不是标注区域）
- `get_label_bbox` 中水平横线方向与斜线x分量一致（315度=右下，横线也向右）
- `process_disease_record` 中用 `cad_x_mid, cad_y_mid` 调用 `get_label_angle`（避免x跨分界线时误判）

### 生成结果
- 全部24页 + 合并文件正常生成，无运行错误

## 下部构件大桩号面剥落病害未绘制（2026-03-24 晚上）
- 问题：盖梁2大桩号面剥落病害（x=6～7.2m, y=0～0.6m, S=0.72m²）未在CAD图中显示
- 根本原因：draw_peel_off函数中的坐标转换错误
  - 错误代码：`center_x = (x1 + x2) / 2 * 10 + origin[0]` （乘以10）
  - 正确代码：`center_x = (x1 + x2) / 2 * SCALE_X + origin[0]` （SCALE_X=14）

- 影响范围：所有剥落/破损病害的绘制位置错误（放大了10/14≈0.71倍），导致图形可能超出边界或位置偏移
- 解决方案：在draw_peel_off函数中，将所有乘以10的地方改为乘以SCALE_X或SCALE_Y
- 代码变更：bridge_disease_main_lower.py 中 draw_peel_off 函数（第315-370行）
- 验证数据：Excel中行123的盖梁2大桩号面剥落病害，之前标注'NG'和'没出来'

## 下部构件避让算法实现（2026-03-24 晚上）
- 问题：两个病害位置很近时，标注会重叠
- 解决方案：
  1. 计算差异化起点：区域病害从右上角，线段病害从左端点
  2. 检测邻近病害：距离阈值3m
  3. 动态调整seg1_len：邻近病害增加50%（裂缝群16→24，其他8→12）
  4. 动态调整seg2_len：邻近病害增加30%
  5. 获取安全角度：避免与邻近病害角度重复，满足上下左右边界约束
  6. 记录角度：供下一个病害避让使用
- 边界约束：
  - 上边界：y <= origin_y (y=0)
  - 左边界：x >= origin_x (x=0)
  - 角度优先级：315°（右下）> 0°（右中）> 270°（下中）> 225°（左下）> 180°（左中）
- 实现文件：bridge_disease_main_lower.py
  - 添加5个辅助函数（calculate_start_point, get_safe_angle, find_nearby_diseases, check_boundary_constraints, draw_disease_label_safe）
  - 修改process_cap_beam_disease支持other_diseases参数
  - 修改所有病害类型的标注逻辑（蜂窝麻面、剥落、网状裂缝、孔洞空洞、侧面、挡块）
  - 主循环传递diseases列表给每个病害
