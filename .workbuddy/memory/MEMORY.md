# 桥梁病害CAD标注系统开发

## 项目概述
根据桥梁病害Excel记录自动生成CAD病害展示图的工具。

### 标注引线参数
- 第一段斜线长度：默认8，网状裂缝/裂缝群使用16
- 第二段横线长度：根据最长文字计算，边距5，最小12
- 上方文字距离横线：0.5倍字高
- 下方文字距离横线：1.5倍字高

### 文件命名格式
- 分页文件：上部T梁第N页_构件号.dxf
- 合并文件：K572+774红石牡丹江大桥（右幅）病害-上部病害.dxf

### 2026-03-23 修改：引线起点计算逻辑优化
问题：剥落露筋等病害的引线起点与实际病害图边界距离过远
解决：修改`draw_peel_off_with_rebar`和`draw_peel_off`函数，支持返回实际边界框(`return_bbox=True`)，`process_disease_record`中使用实际边界框计算引线起点

### 2026-03-24 重构：引线角度逻辑（get_label_angle 完全重写）
新逻辑（两步规则）：
1. **水平方向**：X_CENTER=284，x中点<284→向右(连右侧)，>=284→向左(连左侧)
2. **垂直方向（仅腹板）**：各腹板Y中心线（上部左腹=273.5、右腹=245.3；下部左腹=161.1、右腹=132.9），y中点>中心线→向下，<=→向上
3. 翼缘板/梁底/马蹄：固定方向（左翼缘板/梁底向下，右翼缘板向上，马蹄左向上，马蹄右向下）
4. 组合：右上=45°，左上=135°，左下=225°，右下=315°
同期修复：bbox不含起点、横线方向与斜线x方向一致、用x/y中点计算角度

### 2026-03-24 修改：邻近病害标注重叠处理逻辑
问题：两个病害位置很近时，标注会重叠
解决方案（find_non_overlapping_position函数）：
1. 首先尝试增加seg2_len（横线长度）避免重叠
2. 如果仍重叠，延长seg1_len（第一段斜线长度）尝试分离
   - 尝试 multipliers: 1.5, 2.0, 2.5, 3.0, 4.0, 5.0, 6.0
3. 如果延长seg1后仍重叠，尝试翻转角度（flip_angle）
4. 如果翻转也重叠，尝试翻转+延长seg2
5. 最后尝试其他角度（45/135/225/315）

当两个病害位置太近无法完全分离时，程序会显示警告但仍会生成标注

### 2026-03-25 新增：桥台构件支持 + 单柱墩支持 + 批量处理

在 bridge_disease_main_lower.py 中添加了对桥台和单柱墩构件的支持，并实现了批量处理功能：

**新增功能：**

1. **桥台支持**
   - **模板配置**：
     - 带台身桥台：模板文件 `带台身桥台.dxf`
     - 不带台身桥台：模板文件 `不带台身桥台.dxf`
   - **坐标系统**：
     - 带台身桥台：台帽原点 (66, 245)，台身原点 (66, 219)
     - 不带台身桥台：台身原点 (57, 219)
     - 右下倾斜1度
   - **病害处理**：
     - 新增 `process_abutment_disease()` 函数
     - 支持所有病害类型
     - **只处理台身和台帽，跳过前墙、护栏**

2. **单柱墩支持**
   - **模板文件**：`单柱墩.dxf`
   - **坐标系统**：
     - 盖梁小桩号面原点：(52, 264)
     - 盖梁大桩号面原点：(294, 264)
     - 小桩号面柱子矩形：(143, 233) 到 (167, 194)
     - 大桩号面柱子矩形：(385, 233) 到 (409, 194)
     - 右侧面矩形：(178, 130) 到 (202, 70)
     - 内侧面矩形：(76, 130) 到 (100, 70)
     - 右下倾斜1度
   - **病害处理**：
     - 新增 `process_single_pier_disease()` 函数
     - 处理盖梁和柱子病害

3. **批量处理功能**
   - **上部程序** (bridge_disease_main_upper.py)：
     - 从 `input` 目录读取所有Excel文件
     - 自动处理每个文件的上部病害
     - 生成：`{Excel文件名}-上部病害.dxf`

   - **下部程序** (bridge_disease_main_lower.py)：
     - 从 `input` 目录读取所有Excel文件
     - 自动处理每个文件的下部病害（双柱墩、单柱墩、桥台）
     - 生成：`{Excel文件名}-下部病害.dxf`

4. **文本替换**
   - 路线名称 → 替换模板里的 LLL
   - 桥梁名称 → 替换模板里的 QQQ
   - 构件编号 → 替换模板里的 GGG（盖梁）/ TTT（桥台）

5. **主程序逻辑**
   - 分别处理三种构件类型：双柱墩、单柱墩、桥台
   - 按构件类型和编号生成单独的文件
   - 所有文件合并到最终输出文件

**文件命名：**
- 双柱墩：`下部第N页_盖梁X-墩柱Y.dxf`
- 单柱墩：`单柱墩_X号.dxf`
- 桥台：`桥台_类型_X号.dxf`
- 合并文件：`{Excel文件名}-上部病害.dxf` 或 `{Excel文件名}-下部病害.dxf`
