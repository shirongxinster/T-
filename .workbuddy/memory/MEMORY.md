## 项目概述

### 标注引线参数
- 第一段斜线长度：默认8，网状裂缝/裂缝群使用16
- 第二段横线长度：根据最长文字计算，边距5，最小12
- 上方文字距离横线：0.5倍字高
- 下方文字距离横线：1.5倍字高

### 文件命名格式

- 合并文件：K572+774红石牡丹江大桥（右幅）病害-上部病害.dxf

### 2026-03-23 修改：引线起点计算逻辑优化
问题：剥落露筋等病害的引线起点与实际病害图边界距离过远
解决：修改`draw_peel_off_with_rebar`和`draw_peel_off`函数，支持返回实际边界框(`return_bbox=True`)，`process_disease_record`中使用实际边界框计算引线起点

### 2026-03-24 重构：引线角度逻辑（get_label_angle 完全重写）
新逻辑（两步规则）：
1. **水平方向**：X_CENTER=284，x中点<284→向右(连右侧)，>=284→向左(连左侧)
2. **垂直方向（仅腹板）**：各腹板Y中心线（上部左腹=273.5、右腹=245.3；下部左腹=161.1、右腹=132.9），y中点>中心线→向下，<=→向上
3. 翼缘板/梁底/马蹄：固定方向（左翼缘板/梁底向下，右翼缘板向上，马蹄左向上，马蹄右向下）
4. 组合：右上=45°，左上=135°，左下=225°，右下=315°
同期修复：bbox不含起点、横线方向与斜线x方向一致、用x/y中点计算角度

### 2026-03-24 修改：邻近病害标注重叠处理逻辑
问题：两个病害位置很近时，标注会重叠
解决方案（find_non_overlapping_position函数）：
1. 首先尝试增加seg2_len（横线长度）避免重叠
2. 如果仍重叠，延长seg1_len（第一段斜线长度）尝试分离
   - 尝试 multipliers: 1.5, 2.0, 2.5, 3.0, 4.0, 5.0, 6.0
3. 如果延长seg1后仍重叠，尝试翻转角度（flip_angle）
4. 如果翻转也重叠，尝试翻转+延长seg2
5. 最后尝试其他角度（45/135/225/315）

当两个病害位置太近无法完全分离时，程序会显示警告但仍会生成标注
